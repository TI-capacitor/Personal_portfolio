Automatic Irrigation System

Embedded Systems | PCB Design | Arduino | Bluetooth

Project Overview

Designed and developed an autonomous irrigation controller capable of monitoring environmental conditions and automatically watering crops. The system was intended for home agriculture applications while supporting future deployment in remote locations.

Project Goals

  • Monitor soil moisture levels
  • Monitor environmental temperature
  • Control irrigation automatically
  • Provide Bluetooth communication
  • Support battery-backed operation
  • Prepare for future solar integration

System Architecture

The controller gathers sensor data, evaluates irrigation conditions, and actuates a solenoid valve when watering is required.

Hardware Design

Custom PCB

  • Designed custom PCB using KiCad
  • Considered trace routing and noise reduction
  • Generated manufacturing-ready Gerber files
  • Created complete Bill of Materials (BOM)

Software Architecture

Operating Modes

  • Automatic Mode: Sensor-driven irrigation
  • Maintenance Mode: Disables valve outputs while keeping controller powered

Features

  • Interrupt-driven button interface
  • Bluetooth serial communication
  • USB serial monitoring
  • LCD status display

Engineering Challenges

Display Flickering

Early firmware refreshed the LCD continuously, resulting in visible flickering. The display update logic was redesigned to refresh only when values changed.

Firmware Architecture

The initial design relied on heavily coupled classes. During refactoring, the firmware was reorganized using a state-machine approach to improve maintainability and scalability.

Lessons Learned

  • Importance of modular firmware architecture
  • Benefits of state machines in embedded systems
  • PCB design considerations for mixed-signal systems
  • Designing systems for maintainability

Future Improvements
